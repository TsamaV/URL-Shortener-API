# 🔗 URL Shortener API (Go)

Простой и быстрый сервис для сокращения ссылок на Go.  
Поддерживает регистрацию пользователей, JWT-аутентификацию и сбор статистики кликов.

---

## ⚡ Системные требования

- **Go:** версия 1.21 или выше
- **PostgreSQL:** версия 16 или выше
- **Docker & Docker Compose:** для запуска базы данных
- **ОС:** Windows, macOS, Linux
- **Память:** минимум 32 MB RAM

---

## 📌 Возможности

- 🔐 Регистрация и авторизация пользователей через JWT
- 🔗 Создание коротких ссылок с автогенерацией хэша
- ✏️ Обновление и удаление ссылок
- 📊 Статистика кликов с группировкой по дням или месяцам
- ⚡ Асинхронная запись кликов через in-process Event Bus
- 🛡 Валидация входящих данных и обработка ошибок
- 🌐 CORS и логирование запросов через middleware

## 🚀 Установка и запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/TsamaV/URL-Shortener-API.git
cd URL-Shortener-API
```

### 2. Создать файл `.env`

```env
DSN=host=localhost user=postgres password=my_pass dbname=postgres port=5432 sslmode=disable
SECRET=your_jwt_secret_key
```

### 3. Запустить PostgreSQL через Docker

```bash
docker-compose up -d
```

### 4. Выполнить миграции

```bash
go run migrations/auto.go
```

### 5. Запустить сервер

```bash
go run cmd/main.go
```

Сервер запустится на порту **:8081**

## 📜 API

### 🔐 Аутентификация

#### Регистрация
```
POST /auth/register
```
```json
{
  "email": "user@example.com",
  "password": "secret",
  "name": "John"
}
```

#### Вход
```
POST /auth/login
```
```json
{
  "email": "user@example.com",
  "password": "secret"
}
```

Ответ на оба запроса:
```json
{
  "token": "<jwt-token>"
}
```

---

### 🔗 Ссылки

> Маршруты, помеченные 🔒, требуют заголовка `Authorization: Bearer <token>`

| Метод | Маршрут | Описание | Доступ |
|-------|---------|----------|--------|
| `POST` | `/link` | Создать короткую ссылку | Публичный |
| `PATCH` | `/link/{id}` | Обновить ссылку | 🔒 |
| `DELETE` | `/link/{id}` | Удалить ссылку | Публичный |
| `GET` | `/link?limit=10&offset=0` | Получить все ссылки | 🔒 |
| `GET` | `/{hash}` | Редирект по короткой ссылке | Публичный |

#### Пример создания ссылки
```json
{
  "url": "https://example.com/very/long/url"
}
```

#### Пример получения всех ссылок
```json
{
  "links": [...],
  "count": 42
}
```

### 📊 Статистика 🔒

```
GET /stat?from=2024-01-01&to=2024-12-31&by=day
```

| Параметр | Тип | Описание |
|----------|-----|----------|
| `from` | string | Дата начала `YYYY-MM-DD` |
| `to` | string | Дата конца `YYYY-MM-DD` |
| `by` | string | `day` или `month` |

Ответ:
```json
[
  { "period": "2024-01-15", "sum": 42 },
  { "period": "2024-01-16", "sum": 17 }
]
```

## 🚨 Ограничения

- Требуется активное подключение к PostgreSQL
- Event Bus работает только в рамках одного процесса (не подходит для горизонтального масштабирования)
- JWT не поддерживает отзыв токенов (refresh token не реализован)
- Хэши ссылок генерируются случайно — при коллизии происходит повторная генерация

## ❓ Часто задаваемые вопросы (FAQ)

**Q: Как авторизоваться в защищённых маршрутах?**  
A: После входа или регистрации вы получаете JWT-токен. Передавайте его в заголовке: `Authorization: Bearer <token>`.

**Q: Что происходит при переходе по короткой ссылке?**  
A: Сервер делает редирект `307 Temporary Redirect` на оригинальный URL и асинхронно записывает клик в статистику.

**Q: Почему хэш ссылки иногда генерируется заново?**  
A: Хэш состоит из 6 случайных символов. При совпадении с уже существующим хэшем автоматически генерируется новый — это защита от коллизий.

**Q: Какие коды ошибок могут возникать?**  
A: 
- `401 Unauthorized` — отсутствует или невалидный JWT-токен
- `400 Bad Request` — неверные параметры запроса или тела
- `404 Not Found` — ссылка не найдена
- `500 Internal Server Error` — внутренняя ошибка сервера

**Q: Можно ли запустить без Docker?**  
A: Да, если у вас уже установлен PostgreSQL. Укажите корректный `DSN` в файле `.env` и пропустите шаг с Docker.

**Q: Работает ли на ARM процессорах (Apple M1/M2)?**  
A: Да, Go поддерживает кроссплатформенную компиляцию для ARM.
